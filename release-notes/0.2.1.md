**Patch release. One bug fix.**

## Fixed

- **`otel_genai_graph.__version__` no longer drifts from the wheel's actual
  version.** v0.2.0 shipped with `__version__ == "0.1.0"` hard-coded in
  `__init__.py` even though the wheel itself was correctly tagged
  `0.2.0` — a stale-constant footgun. This release derives `__version__`
  from `importlib.metadata.version("otel-genai-graph")` so the
  import-side version is always identical to the distribution metadata.

  ```python
  # before (v0.2.0)
  >>> import otel_genai_graph; otel_genai_graph.__version__
  '0.1.0'   # but pip show says 0.2.0 — drift!

  # now (v0.2.1)
  >>> import otel_genai_graph; otel_genai_graph.__version__
  '0.2.1'   # always matches pip show
  ```

  A new invariant test (`tests/test_version.py`) locks the import-side
  version against the distribution metadata so this can't drift again.

No API changes. No schema changes. No backend changes. Drop-in over v0.2.0.
